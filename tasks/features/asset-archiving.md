# 기능: 정보자산 증적 시스템

> **목표**: ISO27001/ISMS-P 기준에 따라 회사의 모든 정보자산을 월별로 자동 증적
> **우선순위**: Phase 4 (Supabase 전환 + 자산 유형 확장 후)
> **상태**: 🆕 신규 기능

---

## 1. 개요

### 정의
- **정보자산**: ISO27001 / ISMS-P에서 정의한 조직의 모든 정보 자산
  - 소프트웨어 라이선스
  - 조직 구조 및 조직원
  - 라이선스 할당 및 배정 이력
  - 비용 및 지출 기록

### 증적 방식
- **정기 증적**: 매월 1일 00:00 자동 실행 (지난 1달 데이터)
- **수동 증적**: 관리자가 기간 지정하여 내보내기 (최대 5년)
- **저장소**: 구글드라이브 (카테고리별 폴더, 관리자 설정)
- **형식**: 데이터 정제에 최적화된 형식 (Excel, CSV)

---

## 2. 증적 데이터 구조

### 2.1 포함 항목

#### 라이선스 영역
```
- 라이선스 기본정보 (ID, 이름, 유형, 수량)
- 라이선스 비용 (단가, 총액, VAT, 통화, 환율)
- 할당 현황 (할당 수, 반납 수, 잔여 수)
- 갱신 상태 (예정, 진행중, 미갱신, 갱신완료)
- 담당자 정보 (개인/부서)
```

#### 조직원 영역
```
- 조직원 기본정보 (ID, 이름, 이메일, 부서)
- 조직 배치 정보 (소속 회사, 부서, 직급)
- 상태 (활성, 퇴사유예, 삭제)
- 보유 라이선스 목록
- 마지막 변경일
```

#### 변경 이력 영역
```
- 변경 유형 (추가, 수정, 삭제)
- 대상 엔티티 (License, Employee, OrgUnit, Assignment)
- 변경 전/후 값
- 변경자 (사용자 정보)
- 변경일시
```

### 2.2 비용 계산

#### 공식
```
총액(KRW) = (단가 × 수량 × 환율) × (1 + VAT 포함 여부 ? 0.1 : 0)
```

#### 환율 처리
- 기준: 증적 날짜의 환율 (YYYYMMDD 기준)
- 갱신: 매일 자정 최신 환율로 업데이트 (외부 API)
- 통화: KRW, USD, EUR, JPY, GBP, CNY 지원
- 기본값: KRW 1.0 (고정)

#### VAT 옵션
- 체크박스: "세금 포함" (10% 가정)
- 저장: 라이선스별로 설정 가능

## 3. 자산 카테고리 및 구글드라이브 저장소

### 3.1 카테고리 (관리자 설정)

기본 카테고리:
```
- Software Licenses (소프트웨어 라이선스)
- Infrastructure (인프라/클라우드)
- Hardware (하드웨어)
- Contracts (계약)
- Insurance (보험)
- Other (기타)
```

### 3.2 구글드라이브 폴더 구조

```
Google Drive Root
└── {관리자 설정 폴더1}  (예: License Manager)
    ├── 2026/
    │   ├── 2026-01/
    │   │   ├── 2026-01-asset-full.xlsx  (월간 전체 스냅샷)
    │   │   └── 2026-01-changes.csv       (변경 이력)
    │   └── 2026-02/
    │       ├── 2026-02-asset-full.xlsx
    │       └── 2026-02-changes.csv
    └── Archives/  (수동 내보내기)
        └── 2025-Q4_export.xlsx
```

### 3.3 이메일 공유 (선택)
- 특정 사용자/그룹에 "열람" 권한 부여 가능
- 설정: 카테고리/시간별 선택

## 4. 증적 스케줄

### 4.1 정기 증적 (Scheduled)

**실행 시점**: 매월 1일 00:00 (UTC)

**포함 기간**:
- 시작: 저번 달 1일 00:00
- 종료: 저번 달 마지막날 23:59
- 예: 2월 1일 실행 시 → 1월 1일~31일 데이터

### 4.2 수동 내보내기 (On-demand)

**UI**: `/admin/archives` (ADMIN 전용)

**제약**:
- 최대 범위: 5년 (1825일)
- 결과 형식: Excel (.xlsx) 또는 CSV

## 5. 보안 및 규정 준수

### 5.1 데이터 보호 (ISO27701)
- ✅ 최소 데이터: 필요한 항목만 포함
- ✅ 접근 제어: ADMIN 권한 필수
- ✅ 감사: ArchiveLog 기록
- ✅ 파기: 5년 이후 자동 삭제 옵션

### 5.2 감사 로그 (AuditLog)
- 증적 생성/실패 기록
- 구글드라이브 업로드/공유 기록
- 수동 내보내기 기록

### 5.3 구글드라이브 인증
- OAuth 2.0 (service account)
- 토큰 갱신 자동화
- 실패 알림 (Slack/Email)

## 6. DB 스키마 (Supabase PostgreSQL)

### 필수 테이블
- `ExchangeRate` — 환율 이력
- `AssetCategory` — 관리자 설정 카테고리
- `Archive` — 증적 메타데이터
- `ArchiveLog` — 작업 로그
- `ArchiveData` — 스냅샷 데이터 (선택)

### License 모델 확장
- `isVatIncluded` 컬럼 추가 (10% VAT 옵션)

---

## 7. API 스펙 요약

### 카테고리 관리
- `GET|POST|PUT|DELETE /api/admin/asset-categories/[id]`

### 환율 관리
- `GET /api/admin/exchange-rates` (조회)
- `POST /api/admin/exchange-rates/sync` (외부 API 동기화)

### 증적 관리
- `GET /api/admin/archives` (목록)
- `POST /api/admin/archives/export` (수동 내보내기, 비동기)
- `GET /api/admin/archives/[id]/status` (진행 상태)
- `GET /api/admin/archives/[id]/logs` (작업 로그)
- `DELETE /api/admin/archives/[id]` (삭제)

### 배치
- 정기 증적: 매월 1일 00:00
- 환율 동기화: 매일 09:00
