## 2. 월간 리포팅 상세 프로세스

### 2.1 타임라인

```
매월 1-4일: 데이터 수집 (Security)
         └─ AuditLog 추출
         └─ 백업 로그 확인
         └─ 접근권한 현황 조회

매월 5일: 리포트 작성 및 검토 (Security)
       └─ 자산 현황 분석
       └─ 이상 징후 확인
       └─ 개선사항 정리
       └─ 리포트 최종 검수

매월 6-7일: 배포 및 공유 (Security → Management)
          └─ 경영진 브리핑 (필요 시)
```

### 2.2 수집 쿼리

#### AuditLog 조회
```sql
-- 지난 30일 로그인 이상 (5회 이상 실패)
SELECT actor, COUNT(*) as failed_attempts
FROM AuditLog
WHERE action = 'LOGIN_FAILED'
  AND createdAt >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY actor
HAVING failed_attempts >= 5;

-- 권한 변경 (User role 업데이트)
SELECT actor, details, createdAt
FROM AuditLog
WHERE entityType = 'USER' AND action IN ('UPDATE', 'DELETE')
  AND createdAt >= DATE_SUB(NOW(), INTERVAL 30 DAY);

-- 개인정보 접근
SELECT actor, action, details
FROM AuditLog
WHERE entityType = 'EMPLOYEE'
  AND createdAt >= DATE_SUB(NOW(), INTERVAL 30 DAY);
```

## 3. 분기별 리포팅 상세 프로세스

### 3.1 타임라인

```
분기 마지막 주: 데이터 수집 및 분석 (Security + 자산 책임자)
           └─ 자산 인벤토리 갱신
           └─ 민감도 재평가
           └─ 접근권한 검증

           └─ 3개월 치 데이터 종합 분석
```

## 5. 자동화 기능 (향후 구현)

### 5.1 월간 자동 리포트 생성

**도구**: GitHub Actions / AWS Lambda
**트리거**: 매월 5일 00:00
**작업**:
1. AuditLog에서 지난 30일 데이터 추출
2. SQL 쿼리 실행 (이상 징후 탐지)
3. 리포트 마크다운 자동 생성
4. GitHub에 커밋
5. 팀에 이메일 알림

### 5.2 분기별 자동 체크리스트

**도구**: Tasks 관리 시스템 또는 GitHub Issues
**트리거**: 분기 시작일
**작업**:
1. 자산 갱신 이슈 자동 생성
2. 감사 로그 분석 작업 할당
3. 접근권한 검증 작업 할당

### 5.3 실시간 모니터링 대시보드

**도구**: Grafana / Datadog
**메트릭**:
- 로그인 실패율
- 권한 변경 현황
- 백업 상태
- API 오류율
