# 위협 모델 — License Manager

> 🔒 보안 세션이 관리. 시스템의 위협 지점과 대응 현황을 추적한다.
> 준거: ISMS-P, ISO 27001, ISO 27701, Zero Trust Architecture

---

## 시스템 구성

```
[사용자 브라우저] → HTTP:8080 → [Docker 컨테이너 :3000] → [SQLite 파일]
                                        ↑
                                  EC2 t4g.small (폐쇄망)
```

### 데이터 흐름 상의 신뢰 경계 (Zero Trust)

```
┌─ 신뢰 경계 1: 브라우저 ↔ 서버 ─────────────────────────┐
│  HTTP 통신 (암호화 없음)                                 │
│  위험: 중간자 공격으로 세션 쿠키 / 개인정보 탈취 가능     │
└──────────────────────────────────────────────────────────┘

┌─ 신뢰 경계 2: 서버 ↔ DB ──────────────────────────────┐
│  SQLite 파일 직접 접근                                   │
│  위험: 서버 침해 시 DB 파일 전체 노출                     │
└──────────────────────────────────────────────────────────┘

┌─ 신뢰 경계 3: EC2 인스턴스 ↔ 관리자 ─────────────────┐
│  AWS SSM 접근                                           │
│  위험: SSM 권한 탈취 시 전체 시스템 장악                  │
└──────────────────────────────────────────────────────────┘
```

## 위협 목록

### T-001: 인증 우회
- **대상**: app/api/auth/, lib/auth.ts
- **위협**: 세션 쿠키 위조, 인증 없는 API 접근
- **현재 대응**: 세션 쿠키 + bcryptjs 해싱
- **잔여 위험**: HTTP 환경에서 SECURE_COOKIE=false → 쿠키 탈취 가능
- **근거**: ISO 27001 A.9.4, ISMS-P 2.5
- **Zero Trust 위반**: Verify Explicitly (HTTP로 인해 검증 무력화 가능)
- **상태**: 🟡 수용 (폐쇄망 보완, HTTPS 전환 시 해결)

### T-002: SQL Injection
- **대상**: app/api/ 전체
- **위협**: 사용자 입력을 통한 DB 조작
- **현재 대응**: Prisma ORM (파라미터 바인딩)
- **잔여 위험**: raw query 사용 시 주의 필요
- **근거**: ISO 27001 A.14.2, ISMS-P 2.8
- **상태**: 🟢 대응됨

### T-003: DB 파일 직접 접근
- **대상**: DB 파일 (호스트 경로는 .env.infra 참조)
- **위협**: 서버 접근 시 DB 파일 직접 복사/수정 가능
- **현재 대응**: SSM 접근 제어
- **잔여 위험**: 파일 권한, 백업 미암호화, 무결성 검증 없음
- **근거**: ISO 27001 A.12.3, ISO 27701 8.2.4, ISMS-P 2.9
- **Zero Trust 위반**: Least Privilege (파일 권한 검토 필요)
- **상태**: 🟡 개선 필요

### T-004: Docker 이미지 내 민감 정보
- **대상**: dockerfile, 빌드 산출물
- **위협**: 이미지에 소스코드, 시크릿, 불필요 파일 포함
- **현재 대응**: (확인 필요)
- **근거**: ISO 27001 A.12.6, ISMS-P 2.10
- **상태**: 🔴 미검토

### T-005: XSS (Cross-Site Scripting)
- **대상**: 프론트엔드 전체
- **위협**: 사용자 입력이 HTML로 렌더링되어 스크립트 실행
- **현재 대응**: React 기본 이스케이핑
- **잔여 위험**: dangerouslySetInnerHTML 사용 여부 확인 필요
- **근거**: ISO 27001 A.14.2, ISMS-P 2.8
- **상태**: 🟡 확인 필요

### T-006: 개인정보 비보호 (PII Exposure)
- **대상**: Employee 테이블 (이름, 부서, 직급 등)
- **위협**: 개인정보 평문 저장, 접근 로그 미비, 파기 절차 없음
- **현재 대응**: (확인 필요)
- **근거**: ISO 27701 7.4, 8.2, ISMS-P 3.1~3.5
- **Zero Trust 위반**: Assume Breach (침해 시 PII 전체 노출)
- **상태**: 🔴 미검토

### T-007: 감사 로그 미비
- **대상**: 시스템 전체
- **위협**: 보안 사고 발생 시 추적 불가, 내부자 위협 감지 불가
- **현재 대응**: (확인 필요)
- **근거**: ISO 27001 A.12.4, ISO 27701 8.2.5, ISMS-P 2.11
- **Zero Trust 위반**: Assume Breach (침해 감지·추적 불가)
- **상태**: 🔴 미검토

### T-008: 세션 관리 미흡
- **대상**: lib/auth.ts
- **위협**: 세션 무기한 유지, 동시 세션 미제한, 강제 만료 불가
- **현재 대응**: (확인 필요)
- **근거**: ISO 27001 A.9.4, ISMS-P 2.5
- **Zero Trust 위반**: Verify Explicitly (지속적 검증 미비)
- **상태**: 🟡 확인 필요
