### 4. 에러 로깅

**결과: 🟡 개선 권장**

현재 패턴:
```typescript
console.error("Failed to ...", error);
```

`error` 객체 전체가 로그에 출력되며, 외부 API 호출 에러의 경우 URL, 인증 토큰, 요청 헤더 등이 포함될 수 있음.

**권장사항**:
```typescript
// Before (현재)
console.error("Failed to fetch:", error);

// After (권장)
console.error("Failed to fetch:", error instanceof Error ? error.message : "Unknown error");
```

해당 파일 목록:
- `app/api/licenses/route.ts` — line 77, 187
- `app/api/employees/route.ts` — line 44, 175
- `app/api/assignments/route.ts` — line 19
- `app/api/groups/route.ts` — line 24, 89
- `app/api/assets/route.ts` — 다수
- `app/api/cron/*.ts` — 다수

**위험도**: 낮음 (서버 측 로그에만 노출, 클라이언트에 전달되지 않음)
