# 📋 Asset Manager → Information Asset Platform 비전

> **최종 목표**: 사내 모든 정보자산을 하나의 플랫폼에서 통합 관리하고,
> 월별 자동 증적으로 ISO27001/ISMS-P 준수
>
> **작성일**: 2026-03-07 (Planning Role)
> **상태**: Architecture Design + Phased Implementation

---

## 🎯 사용자의 원래 목표 (2026-03-05)

### 최종 비전
> "내 최종 목표는 회사의 모든 정보 자산을 하나의 플랫폼에서 관리하고, 비용을 추적하고, 매월 1일에 지난 달 자산 기록을 구글 시트나 엑셀 형식으로 저장해서 구글드라이브에 증적까지 하는 것이야"

### 관리 대상 (정보자산)
**ISO27001과 ISMS-P에서 지정한 모든 정보자산**:
- ✅ 소프트웨어 라이선스
- ✅ 클라우드 구독 (AWS, Azure, SaaS 등)
- ✅ 하드웨어 자산 (PC, 노트북, 모니터, 서버, 장비)
- ✅ 도메인 & SSL 인증서
- ✅ 웹 & 애플리케이션 서비스
- ✅ 기타 정보자산
- 🔮 향후: 정책 문서, 준수 아티팩트 (Phase 5)

### 핵심 기능 요구사항

#### 자산 관리
- 모든 자산 유형의 CRUD (등록, 조회, 수정, 삭제)
- 자산 소유권 추적 (담당자, 부서)
- 자산 생명주기 추적 (생성, 활성, 폐기)
- 자산 상태 관리 (활성, 비활성, 폐기 예정)
- 태그 및 분류 시스템

#### 비용 관리
- 자산별 비용 추적 (구입 비용, 월간 비용)
- 부서별/카테고리별 비용 분석
- 월별 비용 추이
- 통화 지원 (환율 자동 동기화 - 일일)
- VAT 계산 (포함/제외 옵션)

#### 소유권 & 할당
- 자산별 담당자 지정
- 부서별 자산 할당
- 할당 변경 이력
- 미할당 자산 추적

#### 모니터링 & 알림
- 만료 예정 자산 알림 (D-70, D-30, D-15, D-7)
- 갱신 일정 추적
- 라이선스 시트 할당 상태 모니터링
- 클라우드 비용 변동 감지

#### 대시보드
- **통합 대시보드**: 모든 자산의 종합 현황
  - 자산 총 개수
  - 월간 총비용
  - 카테고리별 분포
  - 만료 임박 자산
  - 부서별 비용 분포
  - 월별 비용 추이

- **카테고리별 대시보드**: 각 자산 유형별 맞춤 대시보드
  - **소프트웨어**: 라이선스 수, 할당 현황, 갱신 일정, 연간비용
  - **클라우드**: 구독 수, 월간 비용, 사용 현황, 서비스별 분석
  - **하드웨어**: 자산 수, 위치 분포, 수명 주기, 교체 예정
  - **도메인/SSL**: 도메인 수, 인증서 만료 일정, 갱신 비용
  - **웹/앱**: 서비스 수, 운영 상태, 환경별 분포

#### 보고서 & 내보내기
- **월별 종합 보고서**
  - 자산 현황 (타입별, 부서별)
  - 비용 분석 (총액, 월간, 년간 예상)
  - 변동 이력 (신규, 폐기, 갱신)
  - 만료 일정 (다음 30일)

- **Excel 내보내기** (.xlsx)
  - Sheet 1: 자산 현황 (전체)
  - Sheet 2: 변동 이력
  - Sheet 3: 비용 요약 (카테고리별)
  - Sheet 4: 부서별 비용 분석

- **CSV 내보내기**
  - 데이터 정제 형식

#### 월별 자동 증적 (핵심!)
- **스케줄**: 매월 1일 자동 실행
- **대상**: 지난 월의 전체 자산 스냅샷
- **형식**: Excel (.xlsx) 또는 CSV
- **저장처**: Google Drive (자산 카테고리별 폴더)
- **폴더 구조**: `/2026/2026-02/`
- **파일명**: `asset-archive-2026-02.xlsx`
- **포함 내용**:
  - 자산 목록 (현재 상태)
  - 변동 이력 (신규, 삭제, 갱신)
  - 비용 요약
  - 조직원 정보
- **이메일 공유**: 선택적 (관리자, 담당팀)
- **실패 시 재시도**: 자동 (최대 3회)

#### 이력 추적
- **5년 보존**: 모든 자산 변경 이력 (5년)
- **감사 로그**: 누가, 언제, 무엇을 변경했는지
- **스냅샷**: 월별 자산 상태 저장
- **수동 내보내기**: 사용자가 임의의 기간 선택 가능 (최대 5년)

#### 조직 관리
- 회사 (Company) 관리
- 부서 (OrgUnit) 계층 관리
- 조직원 (Employee) 관리
- 부서-자산 매핑

#### 사용자 & 권한
- ADMIN: 모든 기능 접근
- USER: 자신의 자산만 조회/수정
- 역할 기반 접근제어 (RBAC)

#### 규정 준수
- **ISO27001 준수**
  - 정보자산 분류 및 추적
  - 소유권 관리
  - 생명주기 관리
  - 변경 이력 기록

- **ISMS-P 준수**
  - 정보보호 관리 체계 증거
  - 월별 자산 증적
  - 감사 로그
  - 역할 기반 접근제어

## 🔄 진화 로드맵 (5단계)

### Phase 1: 기초 안정화 ✅ (완료)
**목표**: Supabase 전환, 기존 라이선스 시스템 강화

**완료 항목**:
- [x] SQLite → Supabase PostgreSQL 전환
- [x] 감사 로그 시스템 강화
- [x] 입력 검증 추가
- [x] 성능 최적화 (N+1 쿼리, 재귀 쿼리)
- [x] 코드 일관성 (FK 검증, 에러 처리)

**결과**: 견고한 라이선스 관리 시스템 + 확장 가능한 PostgreSQL 기반

---

### Phase 2: Asset 추상화 + 첫 확장 🚀 (다음)
**기간**: 3-4주 | **목표**: License → Asset 모델 일반화, 새로운 자산 타입 추가

**마일스톤**:
- [ ] Asset 기본 모델 설계
- [ ] SoftwareLicense, ITDevice 특화 모델
- [ ] 기존 License → Asset 마이그레이션
- [ ] `/assets` 페이지 구현
- [ ] 카테고리별 CRUD API

**완료 기준**:
- Asset CRUD API 100% 구현
- `/assets/software`, `/assets/devices` 페이지 완성
- License 데이터 Asset으로 성공적 마이그레이션
- 카테고리별 필터링 & 정렬 작동

**팀 작업 분담**:
- **Backend**: Asset 모델, API 구현, 마이그레이션 스크립트
- **Frontend**: `/assets` 페이지, 카테고리 페이지 UI
- **DevOps**: 데이터베이스 마이그레이션 검증

---

### Phase 3: 모든 자산 카테고리 + 통합 대시보드 🚀 (3-4주 후)
**목표**: 모든 자산 타입 추가, 월별 리포트, 대시보드 통합

**추가 자산 타입**:
- [ ] CloudAsset (AWS, Azure, SaaS)
- [ ] WebAppAsset (내부 서비스)
- [ ] DomainSSLAsset (도메인, 인증서)

**주요 기능**:
- [ ] `/assets/cloud` 페이지
- [ ] `/assets/web-app` 페이지
- [ ] `/assets/domains` 페이지
- [ ] 각 카테고리별 특화 대시보드
- [ ] `/dashboard` 통합 대시보드 (모든 자산)
- [ ] `/reports/monthly` 월별 보고서

**완료 기준**:
- 6개 모든 자산 카테고리 CRUD 가능
- 카테고리별 대시보드 표시
- 통합 대시보드 정상 작동
- 월별 리포트 생성 가능

**팀 작업 분담**:
- **Backend**: Cloud/WebApp/Domain API, 대시보드 데이터 API
- **Frontend**: 각 카테고리 페이지, 통합 대시보드, 리포트 페이지
- **Data**: Excel 내보내기 로직
