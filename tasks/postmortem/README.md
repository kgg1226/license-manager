# 포스트모템 시스템 가이드

## 목적

코드 작성 중 발생한 에러를 **카테고리별로 기록**하여 같은 실수를 반복하지 않도록 한다.
모든 Claude 세션(기획/프론트/백엔드)은 작업 시작 전 관련 포스트모템 파일을 읽고, 에러 해결 후 기록한다.

---

## 파일 구조

```
tasks/postmortem/
├── README.md           ← 이 파일 (가이드)
├── docker.md           ← 빌드/Docker 관련
├── db.md               ← DB/Prisma 관련
├── frontend.md         ← 프론트엔드 관련
└── infra.md            ← 배포/인프라 관련
```

## lessons.md와의 관계

| 파일 | 성격 | 기록 시점 |
|------|------|-----------|
| `tasks/lessons.md` | **짧은 규칙** — "~하지 마라", "~해라" | 패턴이 확정되었을 때 |
| `tasks/postmortem/*.md` | **상세 기록** — 증상, 원인, 해결, 예방책 | 에러 해결 직후 |

포스트모템에서 반복되는 패턴이 보이면 → `lessons.md`에 한 줄 규칙으로 승격시킨다.

---

## 세션별 참조 규칙

| 세션 | 시작 시 읽을 포스트모템 |
|------|------------------------|
| 🎯 기획 | 전체 (아키텍처 결정에 영향) |
| 🎨 프론트엔드 | `frontend.md` + `docker.md` (빌드 관련) |
| ⚙️ 백엔드 | `db.md` + `docker.md` + `infra.md` |

---

## 작성 규칙

1. **에러 해결 직후** 바로 기록 (나중에 하면 잊어버림)
2. 각 항목에 **고유 ID** 부여 (PM-D-001 형식)
3. **5가지 항목** 반드시 작성: 증상, 원인, 해결, 예방, 관련 파일
4. 해결되지 않은 이슈도 기록 가능 (상태: 🔧 미해결)
5. 동일 근본 원인의 에러는 기존 항목에 **사례 추가** (중복 항목 금지)
